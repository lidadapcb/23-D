 *      @fft_arr[] 输出 FFT 结果
 *      @fft_N     FFT 点数（支持 1024/2048/4096/8192）

### ✨ 数据流图
```
原始信号 → ADC 采样 → DMA → ADC1_Val[]
       ↓
加窗处理 → 转换浮点 → FFT 运算 → 幅度计算
       ↓
ADC1_FFT[] → 峰值检测 → THD 计算 → 调制识别
```

### ✨ 概念讲解
- **频率分辨率公式**：Δf = f_s / N
- **频点-频率对应**：freq[i] = (i × 500000) / 16384

- **处理时间**：16384 点 FFT ≈ 28ms

## 🔍 关键知识点总结

### 1️⃣ FFT 处理流程
```
原始 ADC 数据 (0~4095)
   ↓ 加窗处理（汉宁窗）
   ↓ 转换为浮点并扩大（×k×10）
   ↓ FFT 变换（Cooley-Tukey 算法）
   ↓ 计算幅度谱（√(real²+imag²)）
   ↓ 频域幅度谱 (8192 个频点)
   ↓ 峰值检测 + THD 计算
```

### 2️⃣ 核心参数
```
采样率：500 kHz
FFT 点数：16384
频率分辨率：30.5 Hz
频谱覆盖：0 ~ 250 kHz
处理周期：约 33ms（采集）+ 28ms（FFT）= 61ms
```

### 3️⃣ 内存分配
```
ADC1_Val[]        32 KB   (12-bit ADC 采样数据)
ADC1_FFT[]       128 KB   (FFT 复数结果)
DSPfft_cache[]    32 KB   (ARM 库工作缓冲)
─────────────────────────
总计             192 KB   (正好填满 F407 SRAM)
```

### 4️⃣ 调试命令

show_status()        - 显示系统状态
